BIG-ser

Sujeito: Sistema de Conexão e Gerenciamento de Clientes com Controle de Acesso

Objetivo:

Implemente um sistema em C que permita a comunicação entre um servidor e vários clientes, onde o servidor tem a capacidade de gerenciar conexões de clientes, bloquear e desbloquear usuários, criar e apagar usuários, e criar APIs que os clientes podem acessar. O servidor deve ter um usuário root com permissões completas para gerenciamento e controle.

Instruções:

1. Estrutura do Sistema:

  • Servidor:
  • Deve aceitar conexões de múltiplos clientes.
  • Deve ter um usuário root que pode executar comandos administrativos (bloquear/desbloquear usuários, criar/apagar usuários, criar APIs).
  • Deve permitir a criação de APIs que clientes podem acessar, com controle de acesso baseado em permissões.

  • Cliente:
  • Deve conectar-se ao servidor e enviar comandos.
  • Deve permitir ao usuário se autenticar e executar ações permitidas com base em suas permissões.

2. Operações Suportadas:

  • Servidor:
  • start_server(port): Inicia o servidor na porta especificada e aguarda conexões de clientes.
  • accept_client(): Aceita uma nova conexão de um cliente.
  • authenticate_user(client_socket, username, password): Autentica um cliente com base no nome de usuário e senha.
  • block_user(username): Bloqueia um usuário para que ele não possa se conectar.
  • unblock_user(username): Desbloqueia um usuário, permitindo que ele se conecte novamente.
  • create_user(username, password, permissions): Cria um novo usuário com permissões específicas.
  • delete_user(username): Apaga um usuário do sistema.
  • create_api(endpoint, handler): Cria uma nova API no servidor.
  • grant_access(api, username): Concede acesso a uma API específica para um usuário.
  • revoke_access(api, username): Revoga o acesso a uma API específica para um usuário.

  • Cliente:
  • connect_to_server(server_ip, port): Conecta-se ao servidor.
  • send_command(command): Envia um comando para o servidor.
  • receive_response(): Recebe a resposta do servidor.

3. Implementação:

  • Servidor:
  • Utilize sockets para gerenciar conexões de clientes.
  • Implemente um sistema de autenticação e autorização para garantir que apenas usuários autorizados (incluindo o root) possam executar comandos administrativos.
  • Mantenha uma lista de usuários, seus status (bloqueado/desbloqueado), e permissões.
  • Implemente a criação e gerenciamento de APIs, garantindo que apenas usuários autorizados possam acessar determinadas APIs.

  • Cliente:
  • Conecte-se ao servidor e implemente uma interface para que o usuário possa digitar comandos.
  • O cliente deve lidar com autenticação e exibir mensagens de erro se o usuário não tiver permissão para executar um comando.


            4. Exemplo de Execução:
            
            - Servidor:
            - Iniciar servidor: `start_server(8080)`
            - Criar usuário: `create_user("john_doe", "password123", "user")`
            - Bloquear usuário: `block_user("john_doe")`
            - Criar API: `create_api("/get_data", handle_get_data)`
            - Conceder acesso à API: `grant_access("/get_data", "john_doe")`
            
            - Cliente:
            - Conectar ao servidor: `connect_to_server("110.0.0.7", 8080)`
            - Autenticar: `send_command("AUTH john_doe password123")`
            - Enviar comando: `send_command("GET /get_data")`
            - Receber resposta: receive_response() // Outputs: "Data: [...]"

5. Restrições:

  • A aplicação deve ser implementada em C e utilizar sockets para a comunicação.
  • O sistema deve ter um controle de acesso robusto para garantir que apenas usuários autorizados possam executar comandos administrativos.
  • A implementação deve permitir a criação e gerenciamento de APIs, bem como o controle de acesso a essas APIs.

Critérios de Avaliação:

  • Correção: O sistema deve funcionar corretamente, com autenticação e autorização funcionando como esperado.
  • Segurança: O sistema deve garantir que apenas usuários autorizados possam executar comandos administrativos e acessar APIs.
  • Eficiência: O servidor deve gerenciar eficientemente múltiplas conexões e operações simultâneas.
  • Organização: O código deve ser bem estruturado, com funções separadas para diferentes operações e um controle claro de permissões.

Submissão:

  • Submeta dois arquivos: ```server.c``` para a implementação do servidor e ```client.c``` para a implementação do cliente.
  • Inclua header no início de cada arquivo.
